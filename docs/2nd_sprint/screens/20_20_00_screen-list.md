## 2. 概要

本ドキュメントは、2ndスプリントで実装する「やりたいこと」管理機能に関連する画面の一覧を記載します。

## 3. 画面一覧

### 3.1 やりたいこと一覧（TodoList）

**画面ID**: `SCR-005`  
**URL**: `/dashboard`（サイドバーで「やりたいこと」を選択）  
**認証**: 必要

#### 目的
ユーザーが「やりたいこと」を管理するための画面。リスト表示、タグフィルター、新規登録、編集機能を提供。

#### 主要な要素

| 要素名 | 種類 | 説明 | 必須 |
|--------|------|------|------|
| ページタイトル | テキスト | 「やりたいこと」 | ○ |
| 新規登録ボタン | ボタン | 新規登録フォームを開く | ○ |
| タグフィルター | UI要素 | タグごとに表示/非表示を切り替え | ○ |
| やりたいことテーブル | テーブル | やりたいことのリストを表示 | ○ |
| 新規登録フォーム | ダイアログ | 新規登録用のフォーム | - |
| 詳細・編集ダイアログ | ダイアログ | 詳細表示・編集用のフォーム | - |

#### テーブルカラム

| カラム名 | 説明 | 表示内容 |
|---------|------|---------|
| No. | 番号 | 連番 |
| タイトル | やりたいことのタイトル | クリック可能（詳細表示） |
| URL | 関連URL | 外部リンクとして表示 |
| ステータス | ステータス | 未着手/進行中/完了 |
| 登録日 | 登録日 | YYYY-MM-DD形式 |
| 更新日 | 更新日 | YYYY-MM-DD形式 |
| タグ | タグ | 複数タグを表示 |
| 優先度 | 優先度 | 高/中/低 |

#### 状態

1. **初期状態**
   - すべてのやりたいことが表示される
   - タグフィルターは未選択
   - 新規登録フォームは非表示

2. **タグフィルター適用時**
   - 選択されたタグを含むやりたいことのみ表示
   - フィルター解除ボタンが表示される

3. **新規登録フォーム表示時**
   - モーダルダイアログで新規登録フォームを表示
   - 背景は半透明のオーバーレイ

4. **詳細・編集ダイアログ表示時**
   - モーダルダイアログで詳細・編集フォームを表示
   - 既存のデータがフォームに表示される

### 3.2 ダッシュボード（ホーム）

**画面ID**: `SCR-002`（更新）  
**URL**: `/dashboard`（サイドバーで「ホーム」を選択）  
**認証**: 必要

#### 変更点（2ndスプリント）

- サイドバーを追加
- 左上の「マイアプリ」をクリックするとホームに戻る機能を追加
- サイドバーで「ホーム」と「やりたいこと」を切り替え可能

---

## 4. 共通コンポーネント

### 4.1 ヘッダー（Header）

- 左上: 「マイアプリ」ロゴ（クリックでホームに戻る）
- 中央: ユーザー名表示（デスクトップのみ）
- 右: ログアウトボタン

### 4.2 サイドバー（Sidebar）

- ホーム: ダッシュボード（ホーム）に遷移
- やりたいこと: やりたいこと一覧に遷移
- アクティブな項目はハイライト表示

---

## 5. データモデル

### 5.1 やりたいこと（Todo）

| 項目 | 型 | 説明 | 必須 |
|------|-----|------|------|
| id | number | 一意のID | ○ |
| title | string | タイトル | ○ |
| url | string | 関連URL | ○ |
| status | string | ステータス（未着手/進行中/完了） | ○ |
| registeredDate | string | 登録日（YYYY-MM-DD） | ○ |
| updatedDate | string | 更新日（YYYY-MM-DD） | ○ |
| tags | string[] | タグの配列 | ○ |
| priority | string | 優先度（高/中/低） | ○ |
| memo | string | メモ | - |
