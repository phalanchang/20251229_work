# 画面一覧

## 1. 概要

本ドキュメントは、ログイン・ログアウト機能に関連する画面の一覧を記載します。
各画面の目的、主要な要素、状態を定義しています。

## 2. 画面一覧

### 2.1 ログインページ（LoginPage）

**画面ID**: `SCR-001`  
**URL**: `/login`  
**認証**: 不要（ただし、ログイン済みの場合はダッシュボードにリダイレクト）

#### 目的
ユーザーが認証情報を入力してログインするための画面

#### 主要な要素

| 要素名 | 種類 | 説明 | 必須 |
|--------|------|------|------|
| ページタイトル | テキスト | 「ログイン」またはアプリケーション名 | ○ |
| メールアドレス入力欄 | 入力フィールド | メールアドレス（またはユーザー名）を入力 | ○ |
| パスワード入力欄 | 入力フィールド | パスワードを入力（マスク表示） | ○ |
| パスワード表示/非表示切り替え | ボタン | パスワードの表示/非表示を切り替え（オプション） | - |
| ログインボタン | ボタン | ログイン処理を実行 | ○ |
| エラーメッセージ表示エリア | テキスト | ログイン失敗時のエラーメッセージを表示 | - |
| ローディングインジケーター | UI要素 | ログイン処理中の表示 | - |
| 入力検証エラーメッセージ | テキスト | 各入力欄の検証エラーメッセージ | - |

#### 状態

1. **初期状態**
   - すべての入力欄が空
   - エラーメッセージは非表示
   - ログインボタンは有効

2. **入力中**
   - ユーザーが入力欄に入力中
   - リアルタイムで入力検証を実行（オプション）

3. **検証エラー状態**
   - 入力検証に失敗した場合
   - 該当する入力欄の下にエラーメッセージを表示
   - ログインボタンは無効化（またはクリック時にエラー表示）

4. **ログイン処理中**
   - ログインボタンをクリック後、APIリクエスト送信中
   - ローディングインジケーターを表示
   - ログインボタンは無効化

5. **ログイン失敗状態**
   - 認証に失敗した場合
   - エラーメッセージ表示エリアにエラーメッセージを表示
   - パスワード入力欄をクリア
   - メールアドレス入力欄は保持
   - ログインボタンは再度有効化

6. **ログイン成功状態**
   - 認証に成功した場合
   - ダッシュボード（または元のページ）にリダイレクト
   - 成功通知を表示（オプション）

#### 入力検証ルール

| 項目 | 検証ルール | エラーメッセージ例 |
|------|-----------|-------------------|
| メールアドレス | 必須、有効なメールアドレス形式 | 「メールアドレスを入力してください」「有効なメールアドレス形式で入力してください」 |
| パスワード | 必須、最小文字数（例：8文字以上） | 「パスワードを入力してください」「パスワードは8文字以上で入力してください」 |

#### アクセシビリティ要件

- すべての入力欄に適切なラベルを設定
- エラーメッセージは適切なARIA属性で関連付け
- キーボード操作で全機能にアクセス可能
- フォーカス順序が論理的

### 2.2 ダッシュボード（DashboardPage）

**画面ID**: `SCR-002`  
**URL**: `/dashboard` または `/`  
**認証**: 必要（未ログインの場合はログインページにリダイレクト）

#### 目的
ログイン後のメイン画面。ユーザーがアプリケーションの主要機能にアクセスするための画面

#### 主要な要素

| 要素名 | 種類 | 説明 | 必須 |
|--------|------|------|------|
| ヘッダー | レイアウト要素 | アプリケーションのヘッダー部分 | ○ |
| ユーザー情報表示 | テキスト | ログイン中のユーザー名やメールアドレス | ○ |
| ログアウトボタン | ボタン | ログアウト処理を実行 | ○ |
| メインコンテンツエリア | レイアウト要素 | アプリケーションの主要コンテンツ | ○ |
| ナビゲーションメニュー | メニュー | 他のページへのリンク（オプション） | - |

#### 状態

1. **初期状態（ロード中）**
   - データ取得中
   - ローディングインジケーターを表示

2. **表示状態**
   - データ取得完了
   - コンテンツを表示

3. **ログアウト処理中**
   - ログアウトボタンクリック後
   - ローディングインジケーターを表示（オプション）

4. **セッション期限切れ状態**
   - セッションが期限切れの場合
   - 自動的にログインページにリダイレクト
   - 期限切れ通知を表示（オプション）

---

### 2.3 保護されたページ（ProtectedPage）

**画面ID**: `SCR-003`  
**URL**: `/protected/*`（例：`/profile`, `/settings`など）  
**認証**: 必要（未ログインの場合はログインページにリダイレクト）

#### 目的
認証が必要な任意のページの例。プロフィールページ、設定ページなど

#### 主要な要素

| 要素名 | 種類 | 説明 | 必須 |
|--------|------|------|------|
| ヘッダー | レイアウト要素 | アプリケーションのヘッダー部分 | ○ |
| ログアウトボタン | ボタン | ログアウト処理を実行 | ○ |
| ページ固有のコンテンツ | レイアウト要素 | ページごとのコンテンツ | ○ |

#### 状態

1. **未認証アクセス時**
   - 認証チェックで未ログインが検出される
   - ログインページにリダイレクト
   - 元のURLを保持（リダイレクト先として）

2. **認証済みアクセス時**
   - 認証チェックを通過
   - ページコンテンツを表示

---

### 2.4 エラーページ（ErrorPage）

**画面ID**: `SCR-004`  
**URL**: `/error` または任意のエラー発生時  
**認証**: 不要

#### 目的
予期しないエラーが発生した場合に表示する画面

#### 主要な要素

| 要素名 | 種類 | 説明 | 必須 |
|--------|------|------|------|
| エラーメッセージ | テキスト | エラーの内容を表示 | ○ |
| ホームに戻るボタン | ボタン | ダッシュボードまたはログインページに遷移 | ○ |
| 再試行ボタン | ボタン | エラーが一時的な場合の再試行（オプション） | - |

## 3. 共通コンポーネント

### 3.1 ヘッダーコンポーネント（Header）

**コンポーネントID**: `CMP-001`

#### 説明
アプリケーション全体で使用するヘッダーコンポーネント

#### 主要な要素

| 要素名 | 種類 | 説明 | 表示条件 |
|--------|------|------|----------|
| アプリケーションロゴ | 画像/テキスト | アプリケーションのロゴ | 常に表示 |
| ユーザー情報 | テキスト | ログイン中のユーザー名 | ログイン済みの場合 |
| ログアウトボタン | ボタン | ログアウト処理 | ログイン済みの場合 |
| ログインボタン | ボタン | ログインページへのリンク | 未ログインの場合 |

---

### 3.2 ローディングコンポーネント（Loading）

**コンポーネントID**: `CMP-002`

#### 説明
処理中の状態を表示するローディングインジケーター

#### 使用箇所
- ログイン処理中
- ログアウト処理中
- データ取得中
- APIリクエスト送信中

---

### 3.3 通知コンポーネント（Notification）

**コンポーネントID**: `CMP-003`

#### 説明
成功メッセージやエラーメッセージを表示する通知コンポーネント

#### 種類
- 成功通知（緑色）
- エラー通知（赤色）
- 警告通知（黄色）
- 情報通知（青色）

#### 表示時間
- 自動的に数秒後に非表示（オプション）
- 手動で閉じることも可能

---

## 4. 画面遷移の概要

### 4.1 認証状態による画面遷移

| 現在の状態 | アクセス先 | 結果 |
|-----------|-----------|------|
| 未ログイン | ログインページ | ログインページを表示 |
| 未ログイン | 保護されたページ | ログインページにリダイレクト（元のURLを保持） |
| ログイン済み | ログインページ | ダッシュボードにリダイレクト |
| ログイン済み | 保護されたページ | ページを表示 |

### 4.2 操作による画面遷移

| 操作 | 現在の画面 | 遷移先 |
|------|-----------|--------|
| ログイン成功 | ログインページ | ダッシュボード（または元のページ） |
| ログアウト | 任意のページ | ログインページ |
| セッション期限切れ | 任意のページ | ログインページ |

---

## 5. 画面設計の原則

### 5.1 レスポンシブデザイン
- モバイル、タブレット、デスクトップに対応
- 画面サイズに応じてレイアウトを調整

### 5.2 アクセシビリティ
- WCAG 2.1 AAレベルを目標
- キーボード操作の完全サポート
- スクリーンリーダー対応

### 5.3 パフォーマンス
- 初回ロード時間の最適化
- 画像の遅延読み込み
- コード分割による最適化

### 5.4 ユーザビリティ
- 直感的なナビゲーション
- 明確なエラーメッセージ
- 適切なフィードバック（ローディング、成功、エラー）
